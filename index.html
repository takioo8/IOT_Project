Advantages

  • Long-range wireless with sub-GHz penetration
  • Ultra-low power — battery life measured in years
  • CSS modulation: immune to narrowband interference
  • High receiver sensitivity (down to −148 dBm)

Limitations

  • Low data rate — not suitable for video/audio
  • Half-duplex only — can't TX and RX simultaneously
  • Duty-cycle restrictions in unlicensed bands (~1%)

Device Classes

CLASS A — ALL

Two RX windows after each uplink. Most power-efficient. Default for sensors.

CLASS B — BEACON

Adds scheduled downlink windows via gateway beacons. Predictable latency.

CLASS C — CONTINUOUS

Always listening. Lowest downlink latency. Highest power draw. For actuators.

Activation Methods

OTAA — Over-the-Air

Secure join procedure. Session keys dynamically derived. Preferred method.

ABP — By Personalization

Static keys pre-flashed. Lower security. Avoid for production.

Security Keys (AES-128)

NwkSKey → Network Session Key (MIC + MAC cmds)
AppSKey → App Session Key (payload encrypt)

QoS Levels

QoS 0 — At Most Once

Fire-and-forget. No ACK. Use for non-critical telemetry.

QoS 1 — At Least Once

Guaranteed delivery, duplicates possible. Use for sensor readings.

QoS 2 — Exactly Once

4-way handshake. No loss or dup. Use for critical commands.

Bottleneck Analysis & Solutions

01

LoRa Duty-Cycle Limit

Unlicensed sub-GHz bands impose ~1% duty-cycle restrictions in Europe, capping transmission frequency. Interference from co-channel devices degrades link quality.

SOLUTION

Use ADR to minimize SF. Switch to event-driven reporting. Minimize payload with binary encoding.

02

Gateway & NS Congestion

A single gateway has finite simultaneous-packet capacity. High SF usage increases time-on-air, crowding out other transmissions. Network Server can be overwhelmed at scale.

SOLUTION

Deploy overlapping gateways. Use Class A devices. Load-balance NS across multiple instances. Minimize downlinks.

03

MQTT Broker Overload

High device count with frequent publishes can saturate a single broker's CPU, memory, and bandwidth — especially with high fan-out to many subscribers.

SOLUTION

Cluster brokers (HiveMQ, EMQ X, AWS IoT Core). Use QoS 0 for non-critical data. Apply broker-side rule engines to filter before fan-out.
